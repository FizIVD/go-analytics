# Высокопроизводительная система сбора событий

Это проект для сбора и обработки большого потока событий от пользователей. Он состоит из:
- **API (api)**: Высокопроизводительный сервис на Go для приема событий.
- **Генератор (generator)**: Утилита для симуляции нагрузки и отправки событий в API.
- **Kafka**: Брокер сообщений для буферизации событий.
- **ClickHouse**: Аналитическая база данных для хранения и анализа событий.
- **Grafana**: Для визуализации данных.

## Оптимизации

- **JSON-сериализация**: В API и генераторе используется `bytedance/sonic`, одна из самых быстрых библиотек для обработки JSON в Go, что значительно снижает задержки при высоких нагрузках.
- **Предварительная компиляция**: Приложения `api` и `generator` компилируются в статичные бинарные файлы внутри Docker-образа. Это позволяет использовать минимальные базовые образы (`alpine`) без Go-компилятора, уменьшая размер и повышая безопасность.
- **Оптимизированный Docker-пайплайн**: Многоэтапные (multi-stage) сборки в `Dockerfile` и использование `.dockerignore` минимизируют размер итоговых образов и ускоряют процесс сборки.

## Запуск

Вся инфраструктура и приложения запускаются с помощью `docker-compose`.

1. **Сборка и запуск всех сервисов (включая API, Kafka, ClickHouse и др.):**
   ```bash
   docker-compose up --build
   ```

2. **Запуск генератора нагрузки:**
   После того как все сервисы успешно запустились (в особенности `api`), откройте новое окно терминала и выполните команду для запуска генератора. Он отправит 1000 "пользователей" в течение 1 "дня" симуляции.
   ```bash
   docker-compose run --rm generator
   ```
   Вы можете настроить параметры генератора, изменив `command` в `docker-compose.yml` или передав их напрямую:
   ```bash
   # Пример: 2000 пользователей за 3 дня
   docker-compose run --rm generator generator -u 2000 -d 3
   ```

## Мониторинг

- **Kafka UI**: Доступен по адресу `http://localhost:8081` для просмотра топиков и сообщений в Kafka.
- **Grafana**: Доступна по адресу `http://localhost:3000` для просмотра дашбордов (требуется настройка).
- **ClickHouse**: Подключение к ClickHouse возможно через клиент или по HTTP-интерфейсу на `http://localhost:8123`.
